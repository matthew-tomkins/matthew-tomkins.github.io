JAVASCRIPT 101

JAVASCRIPT & ITS RELATIONSHIP TO HTML AND CSS

HTML + CSS + JS = the holy trinity of the web

In web development these three core three languages work together as a team - highly complimentary but completely separate. You write them in different files, then have your HTML document call in the other two when a webpage loads.

So if HTML defines what content is on a webpage and CSS makes that content look good... what does JavaScript do?

JavaScript is what brings real interactivity to a web page - it responds to clicks, hovers, form submissions, and can change content on the page without needing to reload. Its main purpose is to bridge the gap between static web content and dynamic user experiences.

CONTROL FLOW AND LOOPS

Flow is how your JavaScript code makes decisions and moves through different paths. Think about making your morning coffee: you check if there's coffee in the machine - if yes, press brew; if no, add coffee first, then press brew. That's control flow - your code following logical decision paths.

Loops are when you need to repeat the same action multiple times. Using the coffee example: imagine you're making coffee for 5 people. Instead of writing "make coffee" five separate times, you'd create a loop that says "make coffee, count one person, repeat until you've made 5 cups." Here's what that looks like in code: 


for (let cups = 1; cups <= 5; cups++) {
    console.log("Making coffee cup number: " + cups);
}

This loop counts from 1 to 5, making a coffee each time without you having to write repetitive code.

ARRAYS VS OBJECTS - ACCESSING DATA

Think of Arrays like a numbered shopping list. Each item has a position (starting from 0), and you access items by their position number: 


let fruits = ["apple", "banana", "orange"];
console.log(fruits[0]); // Gets "apple" (first item)
console.log(fruits[2]); // Gets "orange" (third item)

Objects are more like a contact card where each piece of info has a specific label - technically these are called Keys. You access information by using the label name: 


let person = {
  name: "Sarah",
  age: 25,
  email: "sarah@email.com"
};
console.log(person.name); // Gets "Sarah"
console.log(person.age);  // Gets 25

Arrays are great for lists of similar things (like a shopping list), while objects are perfect for storing related information about one thing (information about a person).

FUNCTIONS (AND WHY THEY'RE USEFUL)

Functions are custom commands you create to do specific jobs. Think of them like creating your own recipe - you write the instructions once, then you can follow that recipe whenever you need it. 


function greetPerson(name) {
    return "Hello, " + name + "! Welcome to our website.";
}

// Now you can use this function anywhere
console.log(greetPerson("Matt")); // "Hello, Matt! Welcome to our website.”
console.log(greetPerson("Sarah")); // "Hello, Sarah! Welcome to our website.”

Instead of writing the greeting code over and over, you write it once in a function and just call it whenever needed. This makes your code way more efficient, easier to maintain, and if you need to change how greetings work, you only update it in one place.

THE DOM AND INTERACTION

The DOM (Document Object Model) is how JavaScript sees and interacts with your webpage. It's basically a family tree that maps out every element on your page - every heading, paragraph, button, and image gets its own spot in this tree structure.

Here's an example: let's say you want to change the text of a button when someone clicks it. JavaScript uses the DOM to grab that specific button element and modify it: 


// Find the button on the page
let myButton = document.getElementById("clickMe");

// Change what happens when it's clicked
myButton.onclick = function() {
    myButton.innerHTML = "Thanks for clicking!";
};

The DOM is the bridge that lets your JavaScript target and manipulate what people see and interact with on your website.


DEVTOOLS - YOUR DEBUGGING COMPANION

DevTools is built into every web browser and it's your best mate for understanding what's happening in your code. You can right-click on any webpage and select "Inspect" to open it up. It lets you see the HTML structure, CSS styling, watch JavaScript run in real-time, and catch errors before they become bigger problems. It's a behind the scenes look at websites.

TO SUM IT UP

JavaScript transforms static web pages into dynamic, interactive experiences. Working in harmony with HTML and CSS, it handles all the logic and interactivity. Through the DOM, JavaScript can manipulate every element on your page, while DevTools gives you the power to debug and refine your code until it works exactly how you want it.
