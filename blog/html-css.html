IDs vs CLASSES

Classes and IDs… what's that about? Here's a quick introductrion to help get your head around things and when to use them.


IDs are super specific and cannot be reused. It's like your home address - it cannot be referenced to navigate anywhere else as a final destination. IDs point to one specific location on the webpage, or a single function.

CLASSES are reusable identifiers that can be applied to multiple elements. They're like suburbs - a shared name that encompasses many different houses within the same area. You use classes to apply consistent styling across elements you want styled the same way throughout your website.


IDS

By nature IDs have significantly more importance than Classes when being processed through the web browser (they have higher CSS specificity), and an ID cannot be reused more than once per web page. IDs are best for targeting specific elements in an HTML webpage, like an anchor tag on a section of a webpage (e.g. 'Our Services').

USE AN ID WHEN:

  • Creating unique page landmarks (navigation, main content, footer)
  • Setting up JavaScript interactions for specific elements
  • Implementing anchor links within a page
  • Targeting elements for automated testing

CLASSES

Classes are broad but specific to their purpose. They're most successfully used for consistent styling and layouts on a web page and can be applied as many times to elements as needed. If you want to style page section headings (e.g. bold, blue & underlined) this can be done in a Class with specific rules defining how it will look.

USE CLASSES WHEN:

  • Styling multiple elements consistently (buttons, cards, text styles)
  • Building reusable component styles
  • Creating modifier variations (.button, .button--large, .button--red)
  • Working with CSS frameworks or design systems

TO SUM IT UP:

Think of IDs as your unique home address and classes as your neighborhood. Use IDs sparingly for one-off elements that need JavaScript interaction or serve as page landmarks. Use classes liberally for styling and creating consistent, reusable components across your site. When in doubt, go with classes - they're more flexible and won't cause headaches down the road, and can be reused as many times as you need.
